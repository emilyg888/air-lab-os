air-lab-os generic engine boundaries — what the engine owns vs what plugins provide

What air-lab-os owns — generic, domain-agnostic
lab/playground.py
Runs one experiment. Calls PatternHandler.run(DatasetHandle) → RunResult. Records to runs.json. Knows nothing about fraud.
lab/arena.py
Compares N patterns on same dataset. Returns ranking. Promotes winner to next tier. Domain-agnostic.
evaluation/evaluator.py
Scores a RunResult against a ScoringPolicy. Reads weights from YAML. Returns composite float. Never hardcodes a metric.
memory/registry.py
Rebuilds pattern state from runs.json. Manages bronze → silver → gold transitions. Persists registry.json.
runtime/llm.py
QWEN plan layer. Reads registry + run history. Returns ranked ExperimentPlan list. No domain knowledge.
runtime/config.py
Loads ScoringPolicy from YAML. Validates weights sum to 1.0. Read-only after load.
What plugins provide — use-case specific
PatternHandler (ABC)
Implements run(handle: DatasetHandle) → RunResult. Fraud detector is one implementation. Any domain provides its own.
DatasetHandle (ABC)
Implements eval_df() → DataFrame, train_df() → DataFrame, meta → dict. bb_datasets is one implementation.
scoring_policy.yaml
Provided per use case. Fraud brings its own weights (f1 40%, expl 25%…). Marketing brings different weights. Engine reads whichever is active.
patterns/scratch|working|stable
Pattern files live here — Python classes implementing PatternHandler. Promoted physically between directories as tier advances.
use-cases/ (Phase 2)
fraud/, marketing/, credit_risk/ — each a self-contained plugin directory with patterns, scoring policy, and dataset handle implementation.
Plugin contracts — the two ABCs air-lab-os defines
PatternHandler — what every pattern must implement
name: str # unique pattern id tier: str # scratch | working | stable run(handle) → RunResult # core method describe() → dict # hyperparams for logging RunResult: flags: list[bool] # per-row prediction scores: list[float] # [0.0, 1.0] confidence explanation: list[str] # "" if no explanation latency_ms: float # set by base class cost_per_1k: float # 0.0 for local
DatasetHandle — what every dataset must implement
eval_df() → DataFrame # fixed 20% eval slice train_df() → DataFrame # fixed 80% train slice labels() → list[bool] # ground truth for eval meta → DatasetMeta # name, domain, tier… schema → dict # column types label_column: str # e.g. "fraud_flag"
